# 📱 响应式布局指南 (Responsive Design Guide)

**项目**: 骑迹大陆 (RideLand)  
**版本**: 1.0  
**更新日期**: 2026-03-20

---

## 🎯 布局策略

### 核心原则

**移动优先 + 响应式适配**

1. **移动端 (≤768px)** - 默认竖屏，符合手机使用习惯
2. **平板端 (>768px)** - 自动适配横屏
3. **桌面端 (>1024px)** - 横屏布局，充分利用屏幕

---

## 📐 断点系统

| 设备类型 | 屏幕宽度 | 布局方向 | 说明 |
|---------|---------|---------|------|
| **手机竖屏** | ≤768px | Portrait | 默认竖屏布局 |
| **手机横屏** | ≤768px | Landscape | 自动适配横屏 |
| **平板** | 769-1024px | 自适应 | 根据方向调整 |
| **桌面** | >1024px | Landscape | 横屏布局 |

## 💻 实现方案

### CSS 媒体查询

```css
/* 手机横屏模式 */
@media (orientation: landscape) and (max-width: 768px) {
  .data-grid {
    grid-template-columns: repeat(4, 1fr);
  }
  
  .page-title {
    font-size: 24px;
  }
}

/* 桌面端横屏布局 */
@media (min-width: 769px) {
  .home-page {
    max-width: 1200px;
    margin: 0 auto;
  }
  
  .data-grid {
    grid-template-columns: repeat(4, 1fr);
  }
}
```

### 屏幕方向检测（可选）

```vue
<script setup>
import { ref, onMounted } from 'vue'

const isLandscape = ref(false)

onMounted(() => {
  // 检测初始方向
  isLandscape.value = window.innerWidth > window.innerHeight
  
  // 监听方向变化
  window.addEventListener('resize', () => {
    isLandscape.value = window.innerWidth > window.innerHeight
  })
})
</script>

<template>
  <div :class="['map-page', { 'landscape': isLandscape }]">
    <!-- 横屏时显示更多地图细节 -->
  </div>
</template>
```

## 📊 布局对比总结

| 页面类型 | 推荐布局 | 理由 |
|---------|---------|------|
| **首页** | 竖屏优先 | 数据卡片纵向排列更紧凑 |
| **地图页** | 横屏优先 | 地图视野更开阔，信息同屏 |

| **骑行页** | 竖屏优先 | 骑行记录时间线，纵向滚动 |
| **排行页** | 竖屏优先 | 排行榜列表，传统纵向 |

## 🎨 设计建议

### 对于地图页面

**强烈推荐横屏布局！** 理由：

1. **地图可视化需要横向空间**
   - 中国地图宽高比约 2:1
   - 横屏可完整显示不缩放

2. **信息密度高**
   - 需要同时显示：地图 + 列表 + 详情
   - 横屏可三分屏布局

3. **符合"战略指挥屏"定位**
   - 横屏更有指挥感、仪式感
   - 类似《文明》《全面战争》等策略游戏

4. **用户行为**
   - 看地图时会自然旋转手机
   - 桌面端用户默认横屏

### 实现建议

```
地图页添加提示：
┌─────────────────────────────┐
│  💡 建议横屏查看以获得     │
│     更好的地图体验          │
│  [旋转手机] [知道了]       │
└─────────────────────────────┘
```

## 📝 总结

**竖屏 vs 横屏的结论：**

| 场景 | 推荐 | 说明 |
|------|------|------|
| **日常数据查看** | ✅ 竖屏 | 手机原生方向，单手操作 |
| **地图疆域** | ✅ 横屏 | 视野开阔，信息同屏 |
| **桌面端** | ✅ 横屏 | 充分利用屏幕宽度 |
| **快速操作** | ✅ 竖屏 | 打开即用，无需旋转 |

**最佳实践：响应式 + 场景化**

- 默认竖屏（移动端）
- 地图页推荐横屏（可提示）
- 桌面端自动横屏
- 添加全屏模式选项
