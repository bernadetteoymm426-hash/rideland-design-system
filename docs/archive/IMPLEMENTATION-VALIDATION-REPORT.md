## 🎯 核心偏差识别

### 偏差 1：缺少"你的家乡"情感定位

**PRD/UX 要求**:
> "用户首次打开看到的不是游戏规则，而是'你的家乡正在被 XX 战队守护'"

**当前实现**: ❌ 缺少自动定位和情感文案

**影响**: P1 原则（情感先于逻辑）未完全实现

**建议修复**:
```vue
// 在 MapView 首次加载时
onMounted(async () => {
  const userLocation = await getUserLocation() // 从顽鹿 App 获取
  highlightTerritory(userLocation.hometown) // 高亮家乡
  showEmotionalMessage(`你的家乡${userLocation.hometown}正在被守护`) // 情感文案
})
```

---

### 偏差 2：缺少周循环状态条

**UX 要求**:
> "顶部常驻周循环状态条，用户随时知道'现在是什么阶段、还剩多少时间'"

**当前实现**: ❌ 缺少顶部状态条

**影响**: P2 原则（一目了然）部分违背

**建议修复**:
```vue
<!-- 添加到 App.vue 或布局组件 -->
<div class="week-status-bar">
  <span class="phase-icon">⚔️</span>
  <span class="phase-name">对战期</span>
  <span class="countdown">距离结算：2 天 14:32:15</span>
</div>
```

---

### 偏差 3：缺少骑行后贡献浮层

**UX 要求**:
> "骑行结束立即看到'你的 23km 推动了攻城进度从 52% 到 58%'"

**当前实现**: ⏳ 待后端联调

**影响**: P0 原则（反馈优先）未完全实现

**建议修复**:
```vue
// 骑行数据提交后
function showContributionFloater(rideData) {
  showModal({
    title: '你的贡献',
    content: `你的${rideData.distance}km 让攻城进度从${progress.old}%推进到${progress.new}%`,
    highlight: rideData.contribution_percentage
  })
}
```

---

### 偏差 4：地图缺少县级细分

**PRD 要求**:
> "全国约 2800 个县区级行政区域"

**当前实现**: ⚠️ 仅演示 5 个城市

**影响**: MVP 完整度不足

**建议修复**:
```js
// 加载完整县级数据
const loadAllCounties = async () => {
  const response = await fetch('https://geo.datav.aliyun.com/areas_v3/bound/100000_full.json')
  const geoJson = await response.json()
  // 提取所有县级区域作为独立领土
  const counties = extractCounties(geoJson)
  initializeTerritories(counties)
}
```

---

### 偏差 5：缺少失败保护机制

**UX 铁律**:
> "永远正面表达"、"失败也有收获"

**当前实现**: ❌ 未实现领土被夺的正面表达

**影响**: 可能导致用户挫败感

**建议修复**:
```vue
<!-- 领土被夺时的文案 -->
<div class="territory-lost">
  <p>虽然失去了{territory.name}，但你的战队成功防守了{defendedCount}个领土</p>
  <p class="positive">本周总骑行量增长了 40%！</p>
</div>
```
